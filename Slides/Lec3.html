Perspective Camera: The Frustum

The Frustum is the geometric shape of the camera's field of vision.

  • Near Plane: Clipping start. Prevents "eye-stabbing" by near geometry.
  • Far Plane: Rendering end. Optimizes performance by ignoring distant pixels.
  • FOV (Field of View): The vertical angle. Wide FOV (90°+) for action, Narrow FOV (30°-) for portraits.

Only objects inside this truncated pyramid are rendered.

Interaction: OrbitControls Mechanism

OrbitControls doesn't move the camera like a car; it moves it like a Satellite.

Target Property

The center of the world for the camera. All rotations happen on a spherical shell centered at controls.target.

controls.target.set(0, 2.5, 0);
controls.update();

Spherical Coordinates

  • Azimuth: Horizontal angle (left/right).
  • Polar: Vertical angle (up/down).
  • Distance: Radius of the orbit.

Interaction Damping (Inertia)

Damping adds physical "weight" to the camera interaction.

MATHEMATICAL FRICTION

Instead of stopping the camera immediately, damping applies a friction coefficient (dampingFactor) to the velocity.

Critical: You must call .update() in the loop!

controls.enableDamping = true;
controls.dampingFactor = 0.05;

function animate() {
  requestAnimationFrame(animate);
  controls.update(); // Recalculates inertia
  renderer.render(scene, camera);
}

Multi-Viewport & Scissor Test

Displaying the POV Preview requires rendering the scene twice on one canvas.

renderer.setScissorTest(true);

// 1. Full Screen Render
renderer.setViewport(0, 0, width, height);
renderer.setScissor(0, 0, width, height);
renderer.render(scene, camera3P);

// 2. PiP Render (Bottom Right)
renderer.setViewport(pipX, pipY, pipW, pipH);
renderer.setScissor(pipX, pipY, pipW, pipH);
renderer.clearDepth(); // Layer POV on top of main scene

The Scissor defines the clipping rectangle, while the Viewport defines the pixel map.

Scene Graph Hierarchy

Our Robot uses a Parent-Child Chain. Transformations flow "downward".

Hierarchy logic:

  • Parent: Joint A.
  • Child: Joint B.
  • If Joint A rotates, Joint B circles around A while maintaining its own rotation.
Track (Group)
 └── RobotRig (Mesh)
      └── PanJoint (Group)
           └── LowerLift (Group)
                └── UpperLift (Group)
                     └── CameraHead (Mesh)
                          └── CameraPOV (Object)

Local Space vs. World Space

Local Space

Coordinates relative to the direct parent.
Ex: head.position.z = 0.6 is constant.

World Space

Coordinates relative to (0,0,0) of the Scene.
Calculated by: ChildMatrix * ParentMatrix.

The cameraPOV doesn't need to know where the robot is; it only cares about its position on the head!

The Smoothness Trade-off

How much data do we need for a 10-second animation?

Stepped (Manual)

Requires 60 snapshots per second (600 total). Huge file size, impossible to edit.

Interpolated (Procedural)

Only needs 2 snapshots (Start/End). Math fills the missing 598 frames automatically.

Interpolation converts "Data" into "Experience".

The LERP Algorithm

Linear Interpolation (LERP) is the core of procedural animation.

Value = A + (B - A) * t
  • A: Starting value (Keyframe 1).
  • B: Ending value (Keyframe 2).
  • t: Progress normalized from 0.0 to 1.0.
// Internal math for t=0.5 (Midpoint)
A = 10; B = 20;
Result = 10 + (20 - 10) * 0.5 = 15;

Normalizing Time with Delta

In our code, we must map Real Time to Normalized Progress (t).

const delta = time - lastTime; // ms since last frame
progressInFrame += delta / frameDuration;

// Clamp at 1.0 and move to next keyframe
if (progressInFrame >= 1) {
  progressInFrame = 0;
  currentIndex++;
}

This ensures the robot moves at the same speed on a 60Hz or 144Hz monitor.

Multi-Parameter Synchronization

The robot doesn't just pan; it moves 7 joints simultaneously using a single t value.

for (let key in robotParams) {
  const startVal = startFrame[key];
  const endVal = endFrame[key];

  // Apply LERP to all joints in sync
  robotParams[key] = startVal + (endVal - startVal) * t;
}
updateRobotVisuals(robotParams);

Beyond Linear: Easing (Intro)

Linear motion is robotic. Real objects have acceleration.

Cubic Easing:
t = t * t * (3 - 2 * t)

By transforming the t value before LERPing, we create "Ease-in-Out" effects.

Observation Lab:

Try toggling interpolation off in the project. The robot "jumps" — this is a t-value of 1.0 instantly.

MediaRecorder API & Canvas Capturing

How we generate the .mp4 file from raw pixels:

  1. Create an Offscreen Canvas for the video frame.
  2. Use ctx.drawImage(webglCanvas, ...) to crop the POV window.
  3. Start mediaRecorder.start().
  4. Push chunks of data into a Blob on every frame.

This happens entirely in the browser — no server required.

Under the Hood: Matrix Math

Three.js objects have a matrixWorld property.

M_world = M_parent_world * M_local

Every time you move a joint, Three.js recomputes these 4x4 matrices for every child in the tree. This is why nested groups are so powerful for Robot Rigs.
